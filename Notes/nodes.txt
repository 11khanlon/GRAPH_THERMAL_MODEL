# ------- Create Node Class ---------
Node represent one graph node 
This is similar to how one block represents one deposition block 
One Node stores: position, temperature, neighbors, activation state 

class Node: 
This initializes the blueprint for the object of node 
"what quantities does each node object contain"

def __init__(self, node_id, position, block_id)
The constructor: This initializes the attributes and variables each node contains
node_id, position, block_id = these are the inputs 
When someone creates a node they must give three inputs 

self.id = node_id  --> this stores the node. Example: node 183 
node_id is a parameter. It only exist while the contructor is running. 
self_id is an instance attribute and belongs to the node object forever 

self.position = np.asarray(position, dtype=float) --> stores (x,y,z)
self.block_id = block_id --> stores what block number its at 
self.active = False --> this means the laser hasn't reached the block yet
self.temperature = MATERIAL["ambient_temperature"] --> directly calls 25 deg C
self.neighbors = []  --> When a node is first created, the graph doesn't exist yet. It will fill later 
self.weights = [] --> Empty for now but will later store the edge weights (heat transfer weights) corresponding to each neighbor

Why doesn't node know geometry?
a node only needs: position, temperature, neighbors 
It doesn't care about STL, layers, build size
